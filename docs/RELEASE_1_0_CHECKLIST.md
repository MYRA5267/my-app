## Открыто — по эпикам Release Roadmap

Ссылки на актуальные чек-листы внутри каждого issue, здесь — только то, что
блокирует именно версию 1.0:

- **#37 Web** — Lighthouse-аудит, кросс-браузерное тестирование (сейчас в основном
  Chromium/Playwright), SEO/Open Graph для расшаривания
- **#38 PWA** — аудит manifest на реальных устройствах, Lighthouse PWA installability;
  сам service worker теперь есть (см. выше), но живой Lighthouse-прогон ещё не сделан
- **#39 Android / #40 iOS** — keystore/сертификаты подписи не заведены, релизные
  сборки только unsigned; см. `docs/MOBILE_PREPARATION.md` за конкретными пробелами
  (кнопка «назад», deep links)
- **#41 Backend** — секреты Production (`SUPABASE_ACCESS_TOKEN`/`SUPABASE_PROJECT_REF`),
  backup-стратегия БД, мониторинг Edge Functions
- **#42 Лицензирование** — файл `LICENSE` **намеренно не создан** этим циклом
  (решение только владельца продукта), аудит лицензий npm-зависимостей, DMCA-процесс
- **#43 Безопасность** — Dependabot подключён, `pnpm audit` чист, финальное
  ревью RLS выполнено; владельцу остаётся включить leaked-password protection
  и GitHub secret scanning
- **#44 Публикация** — версионирование/changelog, чек-лист самого релиза, первые
  сборки в Google Play/App Store — зависят от #39/#40/#42

## Инфраструктурные факты, которые нужно перепроверять при каждом релизе

- `schema.sql` **не** применяется автоматически ни одним workflow — это ручная
  вставка в SQL Editor Supabase (осознанное решение, не забытый шаг)
- `ci.yml` намеренно собирает без Supabase/Sentry секретов — зелёный CI здесь
  проверяет offline-инвариант, а не полную конфигурацию прод-окружения
- Бандл: есть предупреждение Vite про чанки >500kB (`eruda`, dev-консоль) — не
  блокер для 1.0, но следить, чтобы новые тяжёлые ассеты (DETAIL, будущие фичи)
  не попадали в основной чанк без `manualChunks`

## Явно не входит в этот цикл

- Полный `src/features/*` рефакторинг (см. `docs/MYRA_UI_IMPLEMENTATION_PLAN.md`,
  риск №1) — не блокер релиза 1.0, чисто архитектурная работа
- Push-уведомления PWA (Android-системный MediaSession/foreground service уже
  реализован и проверяется отдельным нативным контуром)
