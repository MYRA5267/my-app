# MYRA 1.10.0 — релизный аудит

Дата проверки: 24 июля 2026 года. Ветка: `codex/release-audit-v1`.

## Итог

Код Web/PWA, Android и Windows приведён к единой версии `1.10.0`, собирается
одним lock-файлом pnpm и проходит обязательный автоматический гейт. Текущие
сборки подходят для закрытого тестирования. Публичная публикация в магазинах
пока заблокирована не кодом, а подписью дистрибутивов, юридическими правами на
музыкальный каталог и несколькими настройками владельца инфраструктуры.

## Что проверено автоматически

- `pnpm typecheck` — без ошибок;
- `pnpm test` — 47 из 47 unit-тестов;
- `pnpm build` — production Web/PWA;
- `pnpm test:e2e` — 21 из 21 Chromium mobile E2E-тестов;
- `pnpm audit` — 0 известных уязвимостей;
- Android Gradle — debug APK, unsigned release APK и unsigned release AAB;
- Electron Builder — Windows NSIS setup и portable EXE;
- `git diff --check` — без ошибок пробелов/конфликтных маркеров.

Firefox и WebKit добавлены в smoke-конфигурацию Playwright, но их бинарники не
удалось скачать в текущей среде. Это инфраструктурное ограничение проверки, а
не обнаруженная ошибка приложения.

## Исправления этого прохода

### Безопасность и платежи

- закрыта критическая возможность самостоятельно присвоить подписке
  `active`/`grace` через `set-subscription`;
- production RLS разрешает пользователю только переход собственной подписки в
  `none`; миграция применена к рабочему Supabase;
- `set-subscription` версии 7, `support-chat` версии 11 и `delete-account`
  версии 7 развёрнуты в production с проверкой JWT;
- цена подписки задаётся сервером, а не телом клиентского запроса;
- webhook ЮKassa повторно сверяет платёж с API, суммой, валютой, владельцем и
  метаданными, а также защищён от конкурентной повторной обработки;
- redirect после платежа берётся только из доверенной серверной настройки;
- в Edge Functions добавлены ограничения методов/размера тела и таймауты;
- Electron запрещает Node.js в renderer, webview, небезопасный контент и
  навигацию приложения на внешний origin;
- Android-загрузчик обложек принимает только HTTPS-изображения, ограничивает
  размер ответа и декодирует их с понижением разрешения, чтобы не ловить OOM.

### Надёжность, производительность и интерфейс

- Android MediaSession больше не отправляет фоновому сервису обновление каждую
  секунду; системная шторка получает изменения состояния и экстраполирует время;
- отказ в разрешении уведомлений объясняется пользователю, а не остаётся
  «молчаливой» поломкой мини-плеера;
- Android WebView автоматически использует облегчённую графику без множества
  `backdrop-filter`, тяжёлых blur-слоёв и ненужного PCM-анализа;
- мобильный экран не монтирует скрытое desktop-поддерево;
- прогресс аудио вынесен в узкий контекст: весь экран не перерисовывается каждую
  секунду;
- удалена последняя микроточечная текстура полноэкранного плеера;
- интерактивные карточки получили клавиатурную семантику и доступный фокус;
- стартовый экран стал компактным фирменным сигналом вместо большой заглушки;
- service worker кэширует точные хэшированные Vite-ассеты и даёт рабочую
  offline-оболочку, не перехватывая Supabase, Sentry и платежи.

### Инженерная база

- единственный менеджер пакетов — pnpm 11.9.0, Node.js 22+;
- удалён конфликтующий `package-lock.json`;
- Vite обновлён до исправленной версии 6.4.3;
- транзитивные уязвимости закрыты точечными overrides;
- добавлен Dependabot для npm и GitHub Actions;
- CI теперь выполняет typecheck, unit, build и mobile Chromium E2E;
- добавлены smoke-проекты Firefox и WebKit;
- PWA, Electron и Android собираются из одного production Web-бандла.

## Собранные артефакты

Каталог `release/artifacts` создаётся локально и исключён из Git:

- `MYRA-1.10.0-android-debug.apk` — устанавливаемая тестовая сборка, подписана
  стандартным Android Debug certificate;
- `MYRA-1.10.0-android-release-unsigned.apk` — релизный APK без подписи;
- `MYRA-1.10.0-android-release-unsigned.aab` — пакет для магазина без upload key;
- `MYRA-1.10.0-windows-setup-unsigned.exe` — установщик Windows без Authenticode;
- `MYRA-1.10.0-windows-portable-unsigned.exe` — portable Windows без Authenticode;
- `SHA256SUMS.txt` — контрольные суммы всех файлов.

Unsigned APK/AAB и EXE нельзя выдавать за коммерчески подписанные сборки.
Предупреждения ОС исчезнут только после подписи ключом владельца.

## Что обязан сделать владелец перед публичным релизом

1. Создать и безопасно сохранить Android upload keystore, включить Google Play
   App Signing (или эквивалент выбранного магазина), затем пересобрать AAB.
2. Купить/выпустить Authenticode code-signing certificate и подписать оба EXE
   с timestamp.
3. Подтвердить права на все аудиозаписи, обложки, имена артистов и тексты;
   опубликовать пользовательское соглашение, privacy policy и процедуру
   удаления/жалоб.
4. В Supabase Auth включить защиту от скомпрометированных паролей и проверить
   production redirect URL.
5. Настроить резервные копии, Sentry/алерты Edge Functions и секреты production.
6. Исправить TLS/DNS маркетинговых доменов `myramusic.ru` и `myra-music.ru`.
   Рабочий адрес Web-приложения на момент аудита — `https://app.myramusic.ru`.
7. Выполнить финальный smoke на реальном Android, Windows 10/11, Firefox и
   WebKit/Safari, а также Lighthouse на production URL.

После этих семи действий ветку можно тегировать и публиковать как первый
публичный release candidate.

Supabase Performance Advisor также сообщает только об индексах, которые ещё не
использовались на малом объёме данных. Они намеренно не удалены: до появления
репрезентативной production-нагрузки такой сигнал не доказывает, что индекс
лишний.
