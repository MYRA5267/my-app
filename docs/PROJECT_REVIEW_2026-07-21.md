# MYRA — сквозной ревью проекта (2026-07-21)

React + Vite музыкальный стриминг с опциональным Supabase-бэкендом. Мультиплатформа:
web/PWA, Android/iOS (Capacitor), Electron.

**Вердикт:** кодовая база здоровая и релизо-пригодная для web/PWA Beta. Всё, что осталось
до полноценного мультиплатформенного релиза, — почти целиком нативные сборки и
продакшн-действия владельца, а не код.

Прогнано на ветке `claude/code-review-improvement-59h8yd` (PR #70), ~14 000 строк в 36 файлах
`src/app/`.

## 2. Архитектура и инварианты (по CLAUDE.md) — чисто

- **Optional-backend соблюдён:** рантайм `@supabase/supabase-js` импортит только `supabase.ts`;
  `rooms.ts` берёт лишь `import type { RealtimeChannel }` (стирается при сборке) + клиент из
  `./supabase`. SDK в чужой чанк не течёт.
- **Sentry tree-shaking цел:** `@sentry/react` только в `sentry.ts`, именованный ленивый
  импорт, запрещённого `import * as Sentry` нет.
- **Секретов нет:** `.env` не в git; ни `service_role`, ни приватных ключей в исходниках/
  конфиге. Только плейсхолдеры в `.env.example` и юр.страницах.
- **Чисто по смеллам:** 0 `TODO/FIXME/HACK`, 0 отладочных `console.log`, нет
  `dangerouslySetInnerHTML`.
- Мелочь для доки: формулировку в CLAUDE.md «только `supabase.ts` импортит пакет» стоит
  уточнить как «рантайм-импорт» — `import type` в `rooms.ts` формально исключение, но безвредное.

- **Фаза 15:** постепенная архитектурная чистка / вынос UI-примитивов (низкий риск).
- **Сайт-визитка** (`myramusic.ru`) — черновик готов, ждёт отмашки закоммитить.
- **Фаза 12** (синхронизация между устройствами) — большая фича, выполнима на Supabase,
  нужно согласовать объём.

### Только владелец (BLOCKED_OWNER)

- **Web:** подключить домен `app.myramusic.ru` + DNS, Lighthouse и кросс-браузер/установка PWA
  на реальных устройствах, решение по `noindex`.
- **Android:** keystore и подпись, аккаунты RuStore/Google Play, релизная сборка.
- **iOS:** Mac + Xcode, Apple Developer ($99/год), сертификаты/профили, TestFlight/App Store.
- **Аналитика/платежи:** выбрать провайдера аналитики + политику согласия; продакшн-секреты
  YooKassa при включении платежей.
- **Юр./контент:** заполнить реквизиты на юр.страницах, ревью юриста, финальная модель прав
  каталога.
